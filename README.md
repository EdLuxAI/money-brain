# 搞钱脑 Money Brain

**万物皆可变现的思维引擎** — 一个 Claude Skill，帮你从任何话题中找到可执行的变现路径。

> v0.1 — 经过多轮 Claude × GPT 交叉评审迭代的首个公开版本。

## 这是什么

搞钱脑是一个 [Claude Skill](https://support.anthropic.com/en/articles/claude-ai-skills)，安装后会给 Claude 增加一套商业变现思维框架。它不会让 Claude 变成一个只会谈钱的机器人——而是在你需要商业判断时，提供结构化的分析。

**它做什么：**
- 从话题中识别变现机会，给出具体路径和可行性评估
- 用 9 个思维模型扫描商业机会（信息差、效率差、补贴套利、杠杆、系统化等）
- 每条路径附带 AI 杠杆分析——哪里能用 AI 放大个人产出
- 自动识别你被"白嫖"的风险，给出防御建议
- 对每条路径做七维度可行性评估，附信心标签（已验证/待验证/纯推测）

**它不做什么：**
- 不会劫持你的正常对话（翻译、写代码、聊天等不会被强制商业化）
- 不会编造数字（没有依据时用区间和信心标签，不伪装精确）
- 不会鼓励灰色操作（涉及监管/合规的标记「需专业确认」）
- 不会把所有合作都判定为白嫖（分三级：正常探索/低质量机会/恶意白嫖）

## 核心功能

### 三级响应
不是所有问题都需要咨询报告：
- **快速响应**（3-5句）：「这个能赚钱吗？」→ 直接给方向判断
- **标准分析**：认真讨论商业机会 → 扫描 + 路径设计 + 可行性评估
- **深度分析**：需要执行计划 → 周级规划 + 获客策略 + 定价设计

### 三关检查
每次标准分析前强制过三关：
- Payer 清晰吗？
- 第一单路径清晰吗？
- 合规风险可控吗？

两个以上不通过 → 建议暂停，不硬推。

### 九大思维模型
1. 信息差
2. 效率差
3. 补贴套利
4. 价差套利
5. 注意力模型
6. 杠杆模型
7. 系统化模型
8. 刚需频次矩阵
9. 政策窗口模型

### 白嫖警报系统
识别五种常见白嫖场景，三级分类响应，附反白嫖三道门（Qualification → 收费前置 → 框架不给细节）。

## 安装

### Claude.ai（网页版）

**方法一：直接安装 .skill 文件**
1. 下载 [Releases](../../releases) 中的 `money-brain.skill` 文件
2. 在 Claude.ai 中打开设置
3. 找到 Skills 部分，上传 .skill 文件

**方法二：手动添加**
1. 复制 `money-brain/SKILL.md` 的内容
2. 在 Claude.ai 中创建一个新的 Skill
3. 粘贴内容

### Claude Code（CLI）

**全局安装（所有项目生效）：**
```bash
# 创建 skill 目录
mkdir -p ~/.claude/skills/money-brain

# 下载 SKILL.md（二选一）
# 方式 A：克隆后复制
git clone https://github.com/EdLuxAI/money-brain.git
cp money-brain/money-brain/SKILL.md ~/.claude/skills/money-brain/

# 方式 B：直接下载
curl -o ~/.claude/skills/money-brain/SKILL.md \
  https://raw.githubusercontent.com/EdLuxAI/money-brain/main/money-brain/SKILL.md
```

**项目级安装（仅当前项目生效）：**
```bash
mkdir -p .claude/skills/money-brain
cp /path/to/SKILL.md .claude/skills/money-brain/
```

安装后无需重启，Claude Code 会自动发现并加载 skill。

## 使用方式

安装后正常使用 Claude 即可。技能会在以下场景自动触发：

**强触发**（自动输出商业分析）：
- 你说「这个怎么赚钱」「值不值得做」「谁会买」
- 你描述明确的商业机会或痛点

**弱触发**（末尾附一句提示）：
- 你在聊别的话题，但恰好有商业机会 → Claude 会在回答完后问你要不要展开

**不触发**：
- 纯技术问题、写作、翻译等非商业语境

## 设计理念

这个 Skill 经过多轮迭代，核心设计原则：

1. **先完成主任务**：搞钱视角是附加价值，不抢戏
2. **先过三关再说**：payer、第一单、合规——过不了就不推
3. **不编数字**：没依据就用区间和信心标签
4. **不鼓励灰色操作**：识别政策窗口可以，但涉及合规一律标记
5. **白嫖分级**：不把所有不确定合作都判定为恶意
